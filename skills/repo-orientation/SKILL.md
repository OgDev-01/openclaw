# Repo Orientation

Use this skill when a user wants a quick orientation to a codebase or wants the top-level folders explained before deeper work begins.

## What to do

- Inspect the repository root and identify the main folders and docs.
- Summarize the purpose of each top-level directory in plain language.
- Point out likely entry points for implementation work.
- Call out any obvious workspace conventions or verification scripts.

## Safety notes

- Base the summary on files that actually exist.
- Do not invent project structure that is not present.
- Prefer concise explanations over exhaustive file listings.
